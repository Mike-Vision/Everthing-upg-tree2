1. CORE ANTI-CHEAT REMOTE EVENTS & FUNCTIONS
--------------------------------------------------------------------------------
The game uses a series of RemoteEvents located in `ReplicatedStorage.remotes` to
detect, log, report, and kick players who execute memory injections or exploit 
game mechanics. 

Key security Remotes found in game memory:
* `hyperfail` (RemoteEvent): Triggers an immediate client crash/kick when the game 
  detects critical local variables or memory values being manipulated (e.g., speed,
  jump power, or UI injection).
* `offline_report` (RemoteEvent): Used by local security scripts to send exploit 
  diagnostics and detection parameters directly to the server's telemetry logs.
* `log` (RemoteEvent): Transmits general suspicious activity logs.
* `kick` (RemoteEvent): Server-side command triggered by client verification failure.
* `challenge` (RemoteEvent): Verification challenge used to test client responsiveness.

2. DETAILED BAN & KICK MECHANISMS
--------------------------------------------------------------------------------

A. TELEPORTATION & POSITION CHECKS (Primary Source of Permanent Bans)
- Mechanism: The server runs a continuous loop comparing the player's physical 
  position (Magnitude) over time. If a player teleports instantly across a 
  distance greater than the maximum allowable speed limits, it flags it as a 
  noclip/teleport exploit.
- Trigger: Point Upgrades require teleportation to the corresponding physical board. 
  If the script teleports to boards too rapidly (high frequency), the anti-cheat 
  instantly triggers a server-side ban.
- Trigger (Auto Research): In older versions, Auto Research teleported the player 
  to the Research Center to perform upgrades. Teleporting back and forth between 
  upgrade boards and the Research Center triggered the anti-cheat's teleport checks.
- Resolution in V4.1:
  1. Auto Research has had its teleportation completely REMOVED. Upgrades are now 
     bought remotely while standing completely still, bypassing teleport detection.
  2. Auto Points now utilize a safe distance check (`isClose`) and configurable 
     delays (`UpgradeDelay` defaults to 0.5s) to simulate normal player movement.

B. CLIENT-SIDE TELEMETRY (Diagnostics & Exploit Reports)
- Mechanism: Local scripts (such as `device_detector`, `mouse_behaviour`, or hidden 
  game handlers) monitor environment changes:
  - Injection detection: Checking global tables (`getgenv`, `shared`, `_G`) for exploit UIs.
  - Auto-clicking: Detecting inhumanly fast ClickDetector clicks or Remote fires.
- Action: When an exploit is flagged, the client fires `offline_report` or `hyperfail` 
  to tell the server to permanently flag and ban the account.
- Resolution in V4.1:
  Our script intercepts these reports using a metamethod hook on the game's `__namecall`:
  ```lua
  if (selfName == "diagnostics_event" or selfName == "hyperfail" or selfName == "offline_report" or selfName == "kick") and method == "FireServer" then
      return nil -- Block completely
  end
  ```
  Since the server never receives these report payloads, no automated ban is issued.

C. LOCAL KICK CALLS
- Mechanism: Some local scripts bypass the server entirely and call `Player:Kick()` 
  or index the `.Kick` method directly when a memory error or GUI mismatch occurs.
- Resolution in V4.1:
  We hook both `__namecall` and `__index` metatable methods for `Kick`. Any local 
  kick request is intercepted, logged to the console, and cancelled, allowing the 
  player to continue farming.

D. STAFF / MODERATOR MANUAL DETECTION
- Mechanism: Game developers or moderators manually join public/private servers 
  to inspect players. Teleporting bots are instantly banned manually.
- Resolution in V4.1:
  We implemented an active Moderator Detection loop running every 3 seconds:
  - Scans group ranks for the main group (Id: 33959123). If any player with group 
    rank >= 10 (Mods/Admins) joins, the script immediately logs the player out.
  - Monitors the Workspace for spectator models or camera manipulation, triggering 
    a safe self-disconnect before the mod can issue a manual ban.

--------------------------------------------------------------------------------
3. SAFETY RECOMMENDATIONS FOR PLAYERS
--------------------------------------------------------------------------------
1. Keep `Upgrade Delay` at 0.5s or higher. Lower values increase the risk of 
   speed/teleport flag accumulation on the server.
2. Keep `Min Lambda to Convert` at 10 or higher to reduce convert remote frequency.
3. Keep `Disable Anti-Cheat` and `Anti-Kick` toggles ENABLED at all times.
4. Use private servers when using Moderator Detection, as it prevents staff members 
   from joining and spectating you.
